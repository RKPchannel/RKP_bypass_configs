# ✍️ Справка использования конфигураций:
1. ПРИ ИСПОЛЬЗОВАНИИ КОНФИГОВ НЕ ПЕРЕХОДИТЕ В РОССИЙСКИЕ СЕРВИСЫ, КОТОРЫЕ ВНЕСЕНЫ В БЕЛЫЙ СПИСОК. Лучше выключите VPN, зайдите в сервис, и потом после его использования включите VPN. Иначе вы можете сломать конфиг. Роскомнадзор контролирует поступающий трафик на российские сервисы. Пытаясь войти например в ВК с включенным VPN, вы палите iP конфига, который будет заблокирован, и конфиг перестанет работать по вашей вине.
2. Конфиги были взяты из открытых источников.
3. Конфиги были переименованы для удобства, чтобы понимать, через какой источник будет проходить трафик.
4. Конфиги могут нестабильно работать. Это не моя вина!
5. Конфиги могут быть "фальшивыми", они пингуются у меня и у вас, но при подключении не работать совсем. Это тоже не моя вина.
6. Конфиги могут работать по разному у каждого. Буквально от того к какой вышке вы подключены, заканчивая настройками вашего устройства.
7. Если у вас не работает YouTube, ИИ, InstaGram, и другие зарубежные сервисы, которые заблокированы Роскомнадзором, попробуйте другие конфиги.
8. Я не могу гарантировать полную безопасность конфигов, так как они не мои. Ответственность за кражу и использование ваших данных я не несу.
9. Если у вас из отобранного списка конфигов ничего не работает, попробуйте другие VPN-Клиенты.
10. Если моя сборка конфигураций у вас не заработала, попробуйте другие способы ускорения мобильного интернета в ТГК: **https://t.me/RKP_channel**

# 📋 #RKP_Parser5.0.py - Софт для отбора конфигураций.

С полного нуля мной был создан софт, который автоматизирует поиск и отбор конфигураций. 

Опишу его полную работу:

## 📝 Как работает парсер?

Парсер подключается к Telegram через **MTProto прокси** (для обхода блокировок в России) и запрашивает номер телефона, код подтверждения и пароль двухфакторной авторизации (если включён). Эти данные нужны только один раз – Telethon сохраняет сессию, и при последующих запусках авторизация не требуется.

После успешного подключения парсер:

1. **Проходит по списку Telegram-каналов** (из файла `tg.txt`), читает сообщения, извлекает конфигурации с протоколами **VLESS**, **Trojan**, **Hysteria2**.  
   - Поддерживает прямое извлечение строк, декодирование, скачивание вложенных файлов (`.txt`, `.json`, `.yaml`, `.yml`) и конвертацию их в стандартные URL-форматы.  
   - Обнаруженные в Telegram **HTTP-ссылки на подписки** добавляются в общий список источников для последующей загрузки.

2. **Загружает все найденные источники** (из `sources.txt` и ссылок из Telegram), асинхронно скачивает их и извлекает конфиги.  
   - Поддерживаются прямые ссылки, **base64**, **JSON-конфиги Xray** и **YAML-конфиги Clash / Sing-box**.

3. **Добавляет ручные конфиги** из `my_configs.txt` – они проходят все последующие этапы обработки.

4. **Очищает дубликаты и невалидные конфиги** – удаляются битые строки, синтаксически некорректные URL, повторы (нормализация параметров запроса).

5. **Фильтрует конфиги по IP и SNI**:  
   - Благодаря первым двум октетам iP и файлу с SNI, которые работают при белом списке собираются конфиги для сборки подписки чтобы обходить белые списки.
   - Остальные конфиги отбрасываются для сборки конфигов в черный список.

6. **Очищает названия** – удаляет всё, что было после `#`, оставляя пустой тег. Это необходимо для стабильной работы парсера.

7. **TCP-проверка** – быстро проверяет **только VLESS и Trojan** конфиги на открытый порт (таймаут 12 секунд). Hysteria2 пропускаются без этой проверки (так как их работоспособность эффективнее проверять полным тестированием).

9. **Полноценная проверка на работоспособность** – запускаются ядра **Xray** и **Hysteria2**.  
   - Для каждого конфига создаётся временный конфиг, ядро слушает локальный SOCKS5-порт.  
   - Через этот порт отправляется запрос к тестовому URL (`https://www.gstatic.com/generate_204`).  
   - Если получен ответ 200 или 204, конфиг считается рабочим, измеряется пинг.  
   - Поддерживаются **все транспорты**: `tcp`, `ws` (WebSocket), `grpc`, `httpupgrade`, `xhttp`, `h2`, `kcp`, `quic`, а также **TLS** и **Reality** с настройками `sni`, `fingerprint`, `alpn`, `allowInsecure`.  
   - Проверка параллельная, что позволяет быстро обработать тысячи конфигов.

10. **Переименование и кодирование** – каждому рабочему конфигу добавляется название со страной и подписью `[#РКП]`. Тег кодируется через URL-encoding, чтобы избежать проблем с символами.

11. Результаты сохраняется в различные форматы для дальнейшего использования)

14. **Цикл повторяется через 1 час**.  
    - Перед новым циклом парсер **импортирует уже работающие конфиги** в новый цикл - это позволяет не терять их, даже если все источники временно недоступны.

## 🛡️ Устойчивость к сбоям
- **Антисон** – если операция с Telegram зависает более 120 секунд, парсер перезапускает прокси (до 3 раз).  
- **Обработка FloodWait** – при получении ошибки `FloodWait` от Telegram парсер **немедленно прекращает** парсинг каналов, сохраняя уже собранные конфиги, и переходит к источникам.
- **Обработка ошибок соединения** – при ошибке `server closed the connection` парсер пытается переподключиться до 3 раз, затем пропускает Telegram-парсинг.  
- **Корректное завершение** – все временные файлы и процессы ядер удаляются/убиваются после проверки.

## 🔌 Поддержка протоколов и транспортов
- **VLESS** (с Reality, TLS, xtls-rprx-vision, flow)  
- **Trojan** (с любым паролем, включая символ `#` – исправлена проблема с `%23`)  
- **Hysteria2** (с auth, obfs-salamander, bandwidth)  
- Транспорты: tcp, ws, grpc, httpupgrade, xhttp, h2, kcp, quic.  
- Параметры: sni, host, path, serviceName, alpn, fingerprint, allowInsecure, publicKey (pbk), shortId (sid), spx, extra и другие.

## 🖥️ Требования к системе
- **Windows, Linux, MacOS** (проверено на Windows 10)  
- **Python 3.8+**  
- **Установленные зависимости** (см. в zip-архиве с паресром)  
- **Бинарные файлы ядер** (`xray.exe`, `hysteria2-windows-amd64.exe`) в папке `cores`  
- **API ID / Hash Telegram** (получить на (https://my.telegram.org))  
- **MTProto прокси на TG WS Proxy** – для работы в России
