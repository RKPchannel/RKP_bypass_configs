# 📋 #RKP_Parser4.0.py - Софт для отбора конфигураций.

С полного нуля мной был создан софт, который автоматизирует поиск и отбор конфигураций. 

Опишу его полную работу:

## 📝 Как работает парсер?

Парсер подключается к Telegram через **MTProto прокси** (для обхода блокировок в России) и запрашивает номер телефона, код подтверждения и пароль двухфакторной авторизации (если включён). Эти данные нужны только один раз – Telethon сохраняет сессию, и при последующих запусках авторизация не требуется.

После успешного подключения парсер:

1. **Проходит по списку Telegram-каналов** (из файла `tg.txt`), читает **до 3000 последних сообщений** на канал, извлекает конфигурации с протоколами **VLESS**, **Trojan**, **Hysteria2**.  
   - Поддерживает прямое извлечение строк, декодирование **base64**, скачивание вложенных файлов (`.txt`, `.json`, `.yaml`, `.yml`) и конвертацию их в стандартные URL-форматы.  
   - Обнаруженные в Telegram **HTTP-ссылки на подписки** добавляются в общий список источников для последующей загрузки.

2. **Загружает все найденные источники** (из `sources.txt` и ссылок из Telegram), асинхронно скачивает их (35 одновременных потоков) и извлекает конфиги.  
   - Поддерживаются прямые ссылки, **base64**, **JSON-конфиги Xray** и **YAML-конфиги Clash / Sing-box**.

3. **Добавляет ручные конфиги** из `myurl.txt` – они проходят все последующие этапы обработки.

4. **Очищает дубликаты и невалидные конфиги** – удаляются битые строки, синтаксически некорректные URL, повторы (нормализация параметров запроса).

5. **Фильтрует конфиги по SNI**:  
   - Пропускаются только те, у которых параметр `sni` (или `host`) **заканчивается на `.ru`, `.рф`, `.xn--p1ai`**  
   - **Или** если домен входит в список разрешённых из файла `domains.txt` (можно задать любые домены самостоятельно).  
   - Остальные конфиги отбрасываются – это обеспечивает релевантность для российских пользователей.

6. **Очищает названия** – удаляет всё, что было после `#`, оставляя пустой тег. Это необходимо для стабильной работы парсера.

7. **Исправляет параметры `spx` и `extra`** в VLESS-ссылках – выполняет многоуровневое декодирование, исправляя случаи двойного кодирования, которые встречаются в некоторых публичных подписках.

8. **TCP-проверка** – быстро проверяет **только VLESS** конфиги на открытый порт (таймаут 12 секунд). Trojan и Hysteria2 пропускаются без этой проверки (так как их работоспособность эффективнее проверять полным тестированием).

9. **Полноценная проверка на работоспособность** – запускаются ядра **Xray** и **Hysteria2**.  
   - Для каждого конфига создаётся временный конфиг, ядро слушает локальный SOCKS5-порт.  
   - Через этот порт отправляется запрос к тестовому URL (`https://www.gstatic.com/generate_204`).  
   - Если получен ответ 200 или 204, конфиг считается рабочим, измеряется пинг.  
   - Поддерживаются **все транспорты**: `tcp`, `ws` (WebSocket), `grpc`, `httpupgrade`, `xhttp`, `h2`, `kcp`, `quic`, а также **TLS** и **Reality** с настройками `sni`, `fingerprint`, `alpn`, `allowInsecure`.  
   - Проверка параллельная (50 потоков), что позволяет быстро обработать тысячи конфигов.

10. **Переименование и кодирование** – каждому рабочему конфигу добавляется тег вида `SNI: <домен> [#РКП]`. Тег кодируется через URL-encoding, чтобы избежать проблем с символами.

11. **Сортировка** – итоговый файл строится в порядке: сначала все VLESS, затем Trojan, затем Hysteria2 (каждый протокол отсортирован по алфавиту).

12. **Принудительное добавление конфигов** – из файла `forced.txt` (только VLESS) конфиги добавляются в финальный список, даже если они не прошли проверку. Это полезно для «запасных» или особо надёжных серверов.

13. **Результат сохраняется в `url_work.txt`** – именно этот файл используется для повторной проверки в следующем цикле и для отправки в GitHub/Telegram.

14. **Цикл повторяется через 30 минут** (настраивается параметром `CYCLE_DELAY`).  
    - Перед новым циклом парсер **импортирует уже работающие конфиги** из `url_work.txt` – это позволяет не терять их, даже если все источники временно недоступны.

## 🛡️ Устойчивость к сбоям
- **Антисон** – если операция с Telegram зависает более 120 секунд, парсер перезапускает прокси (до 3 раз).  
- **Обработка FloodWait** – при получении ошибки `FloodWait` от Telegram парсер **немедленно прекращает** парсинг каналов, сохраняя уже собранные конфиги, и переходит к источникам.
- **Обработка ошибок соединения** – при ошибке `server closed the connection` парсер пытается переподключиться до 3 раз, затем пропускает Telegram-парсинг.  
- **Корректное завершение** – все временные файлы и процессы ядер удаляются/убиваются после проверки.

## 🔌 Поддержка протоколов и транспортов
- **VLESS** (с Reality, TLS, xtls-rprx-vision, flow)  
- **Trojan** (с любым паролем, включая символ `#` – исправлена проблема с `%23`)  
- **Hysteria2** (с auth, obfs-salamander, bandwidth)  
- Транспорты: tcp, ws, grpc, httpupgrade, xhttp, h2, kcp, quic.  
- Параметры: sni, host, path, serviceName, alpn, fingerprint, allowInsecure, publicKey (pbk), shortId (sid), spx, extra и другие.
